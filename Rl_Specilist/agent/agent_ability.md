agent 所需要具备的能力：
1. 严谨性问题， 任何稍微稍微的不对头， 都会引起错误， 强调逻辑完整性。 
2. 韧性问题， 对于调用工具失败的情况， 会尝试几次。 
3. 灵活性问题， 对于某种工具调用， 或某种路径已经产生了死胡同时， 能够换一种方式。 
4. 简单的工具调用。 知道哪种类型问题调用哪种工具。 
5. 格式问题， 先大致了解一件事情后， 再开始做事情， 然后边做边修正。 
6. 效率问题。 在了解一些事情之后， 不再调用工具， 而是自身能够产生深入的思考和模拟， 然后能够剪枝探索， 减少工具的调用。 
7. 实际做事情的能力，怎样写代码， 怎样做表格， 做ppt， 实际的技能。 
8. 长程目标一致性。 能够在工作很久之后， 仍然记住之前的目标， 并且清晰了解当前所处的位置。 
9. 拆解复杂任务能力。 能够把一个复杂的规划， 拆分成N个小点， 然后再逐步完成
10. 对全局规划的能力， 不只是局限于局部进行规划和解决问题， 要有全局意识。
11. 能够坚定事实的能力 。 用户发送错误认知和指令时， 要能够坚持自己， 纠正用户。
